  I think that the reason people don’t like change is not because they are not willing to try something new (or something better), it’s because there are difficulties, or “growing pains”, that accompany trying something new.
  Over the past week, I’ve been using the IntelliJ Idea IDE and it definitely has some growing pains. For example, Installing the software itself and disabling all of the plug-ins and extra software was an hour long process itself. Furthermore, loading the “.eslintrc” and the “package.json” files into the directory whenever creating a new project seems like something I would be required to do in 2003, not in 2021. Lastly, the eslint code checking software is much slower than that of other IDE’s I’ve used in the past. Sometimes it will take up to 7 seconds for a green checkmark to appear at the top right of the screen (indicating that there are no codestyle errors in the source code) after fixing all of the errors.
  However, with these growing pains come a few triumphs. Although I’ve only been using IntelliJ Idea for a week, I’m already starting to anticipate some frequent codestyle errors I’ve been making over the past week such as neglecting to type spaces around math operators. I also find the ease of alternating between HTML and Javascript project windows to be extremely convenient for web development. Lastly, when building an HTML project in IntelliJ Idea, the IDE gives the user the choice of opening the project in multiple different browsers like Chrome or Opera. Opening a project in multiple different browsers was a task I found very tedious when first learning HTML on the Notepad++ IDE.
  To summarize, after the first week of using IntelliJ Idea, I think it’s a powerful IDE that will be beneficial to my overall learning experience, but won’t be without its inconveniences.
